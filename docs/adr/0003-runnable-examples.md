# Runnable examples as workspace packages

The original usage-examples PRD scoped the `examples/` directory as illustrative-only: single TypeScript files, no `package.json`, no build step, "runnable dev server or build tooling" explicitly out of scope. Revisiting that decision, we now want each example to be a fully runnable standalone web app that a developer can clone and start with one command — the inert-snippet shape was too far from what someone evaluating Leitner actually needs.

Decision: each subdirectory under `examples/` becomes a self-contained Vite project (its own `package.json`, `index.html`, `vite.config.ts`, source files) and the repo root becomes an npm workspaces root with `"workspaces": ["examples/*"]`. Each example references the library as `"leitner": "file:../.."` — npm 7+ links `file:` deps as a symlink under `node_modules/leitner`, and the root's `exports` map gates access to `dist/` only, so the React example's `useSyncExternalStore` typing comes from the same `dist/index.d.ts` a real consumer would see. Each example carries a `"predev"` script that runs `npm --prefix ../.. run build` so a single `npm run dev` builds Leitner and starts Vite in one step. The workspaces root is what gives us the single-`npm install`-at-root install topology; `file:../..` is what wires the library into the example because npm does not make a workspaces root a member of its own workspaces (so `"leitner": "*"` would miss locally and 404 on the registry).

Two alternatives were rejected. A Vite alias from `'leitner'` to `../../src/index.ts` would silently bypass the package's `exports` map, masking broken exports that real consumers would hit. Pinning a published version (`"leitner": "^0.1.0"`) would let examples lag the working tree until a release. The cost of the workspaces choice is that root `npm install` now also installs Vite, React, and react-dom into `node_modules`, and the examples can drift between releases — a deliberate trade-off; the prepublish smoke check is intentionally not extended to build the examples (the reader is the canary), keeping `prepublishOnly` lean.
